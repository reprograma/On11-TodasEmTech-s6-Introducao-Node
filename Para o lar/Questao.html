1 - Qual a relação entre os métodos HTTP e o CRUD?

Muitas vezes um CRUD é confundido com uma RESTFul, porém eles não são a mesma coisa, essa confusão se dá pelo fato de que os métodos HTTP são semelhates e dá para confundir com uma ligação direta.

O CRUD são as siglas de CREAT, READ, UPDATE E DELETE e são utilizadas na referencia das operações que são realizadas no banco de dados relacional sendo elas consulta, inclusão, alteração e exclusão dos registros.

Os métodos HTTP são métodos de requisição responsáveis por indicar a ações que serão executada para determinados recursos. Os primcipais métodos HTTP são: POST, GET, PUT, PATCH, DELETE.

A relação se dá devido a semelhança de classificação

  • CREAT -- POST
  • READ -- GET
  • UPDATE-- PUT/PATCH
  • DELETE -- DELETE

2 - Comente, com exemplos, a diferença entre o PUT e o PATCH

Ambos os métodos são usados para indicar a requisição de uma alteração a diferença entre eles é que o método PUT realiza a alteração completa do seu recurso e possibilita a criação do recurso caso o mesmo não exista enquanto que o PATCH realiza a alteração parcial do recurso e caso não exita não é possivel cria-lo

Exemplos praticos de PUT e PATCH

Em um banco de dados de Professores eu tenha os dados de id, nome, disciplina, colegiado e departamento para fazermmos alterações utilizando PUT eu preciso passar todos os dados do recurso preenchido, ou seja seu eu quiser alterar somente o departamento do professor ou criar um novo recurso precisarei passar todos os demaias dados também, id, nome, disciplina, colegiado e departamento. Para que eu consiga fazer a alteração apenas de um dado já existe do recurso usa-se o método PUTCH

4 - Defina o conceito de idempotência e como uma API pode ser idempotente

Idempotencia para a matematica e a ciência da computação é a propriedade que uma operação têm de poderem ser aplicadas varias vezes e não sofrer alteração no seu resultado. Ao relacionarmos com API's temos que uma API é idempotente quando um método é idempotente ou seja, quando o resultado de uma requisição com sucesso é o mesmo independente das vezes que ela foi executada seus requisitos não são alterados um exemplo de metodo idempotentes GET, PUT, DELETE, como o método POST é de criar um recurso ele não pode ser um método idempotente
