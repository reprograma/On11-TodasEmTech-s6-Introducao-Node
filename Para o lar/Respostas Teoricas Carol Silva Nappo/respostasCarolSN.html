Olá, para explicar a relação entre os metodos HTTP e CRUD, vamos primeiro entender um pouco sobre cada um: 

CRUD é o acrônimo em inglês, tipo uma sigla, sabe? Ou seja:

C create (criar)
R read (ler)
U Update (atualizar)
D Delete (excluir)

O CRUD é uma espécie de padronização de construção, ou um estilo de arquitetura, usadas em base de dados.  

Já o HTTP, como vimos na aula é um protocolo de comunicação da web para enviar e receber pedidos (requests) do cliente ao servidor, que por sua vez devolve respostas (responses). Para acontecer essa sintonia ai cliente/servidor, existe uma série de métodos (ou verbos) que podemos usar. Os mais conhecidos são:

GET ler
POST criar
PUT substituir
PATCH modificar
DELETE excluir

E então, viu ai algumas relações das ações de CRUD com os métodos de HTTP ? 

De maneira muito simplista, podemos chamar o servidor de base de dados, certo? Então aí está: a maneira como os métodos HTTP funcionam, podem ser consideradas operações do modelo CRUD, entendeu?


Eles são métodos de requisição do HTTP. Que nada mais significa do que dizer que são comandos para fazer algo naquela comunicação entre o cliente e o servidor.

De acordo com a definição do nosso querido developer.mozilla.org eles são responsáveis por:

O método PATCH é utilizado para aplicar modificações parciais em um recurso.
O método PUT substitui todas as atuais representações do recurso de destino pela carga de dados da requisição.


Ok, então ambos servem para atualização de conteúdo. E...?
Bom, para ajudar na compreensão, acho sempre interessante pensar na tradução da palavra para o português, de forma literal: PUT significa "colocar" e PATCH significa "remendar".
Daí ja notamos que o PATCH é uma ação menos completa e a PUT mais "intensa", por assim, dizer.
Assim como na vida, se não queremos gastar energia (no caso, dados) de forma excessiva, qual seria o melhor método? O PATCH, né?

Um último exemplo, não tecnicamente correto, mas mais didático, seria:
Se um usuário preencheu um formulário ontem e hoje fizemos uma atualização e pedimos para preencher novamente, aconteceria o seguinte:
PUT - seria criado um novo cadastro
PATCH - somente os novos dados inseridos serão enviados e salvos.

Tendeu?

Lembrando que o metodo PUT é necessariamente, idempotente, já o PATCH, não necessariamente. O conceito de idempotência será melhor abordado na questão 4.
