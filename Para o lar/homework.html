1) Qual a relação entre os métodos HTTP e o CRUD?

Pode-se dizer que HTTP é um protocolo que determina os padrões e define quais serão as regras relacionadas as trocas de informações entre os servidores que hospedam os sites e os computadores. Ele é responsável por organizar o tráfego de informaões além de realizar a codificação de forma que o site chegue até o usuário quando este tentar acessá-lo.

Já o CRUD pode ser definido como um conjunto de operações primitivas, bastante utilizado em banco de dados e armazenamento de dados estatísticos. É responsável por realizar as operações em um repositório de dados. O CRUD é um acrônimo para as 4 operações básicas de um banco de dados: create (insert), read (select), update (update) e delete (delete).

A relação do HTTP e do CRUD é na hora do recurso. Uma vez que o recurso é estabelecido, será necessário identificar quais as ações serão aplicadas ao recurso e como eles serão alocados no API. O princípio do RESTFULL fornece meios para lidar com as ações do CRUD usando os métodos HTTP. Exemplo: GET , POST, PUT, PATCH e DELETE.

2) Comente, com exemplos, a diferença entre o PUT e o PATCH.

O método PUT aplica uma atualização completa do recurso além de ser idempotente, já o PATCH atualiza apenas um subconjunto dos dados do seu recurso e não é idempotente. No PUT para que a alteração seja realizada, será necessário enviar todos os parâmetros da entidade, o que não é necessário quando se utiliza o PATCH. Exemplificando: { "username": "raquel", "email": "raquel@hotmail.com" }, ao fazer um PUT para atualização do e-mail, Temos { "username": "raquel", "email": "raquel@hotmail.com" } e o PATCH fica sendo: { "username": "raquel@gmail.com"}.

4) Defina o conceito de idempotência e como uma API pode ser idempotente.

A idempotência é uma propriedade que algumas operações tem de poder ser aplicadas várias vezes sem que o valor do resultado seja alterado. Exemplos de verbos idempotentes: GET, PUT, DELETE, HEAD e OPTIONS. A idempotência garante que uma solicitação de API seja concluída apenas uma vez. Com uma solicitação idempotente, se a solicitação original for concluída com êxito, as novas tentativas subsequentes retornarão o resultado da solicitação original bem-sucedida e não terão efeito adicional.

5) Cite alguns diferentes padrões de projetos de software

Existem em torno de 23 padrões, divididos nas seguintes três categorias:

  1. Criacionais:padrões que propõem soluções flexíveis para criação de objetos. São eles: Abstract Factory, Factory Method, Singleton, Builde, Prototype.
  2. Estruturais padrões que propõem soluções flexíveis para composição de classes e objetos. São eles: Proxy, Adapter, Facade, Decorator, Bridge, Composite e Flyweight.
  3. Comportamentais:padrões que propõem soluções flexíveis para interação e divisão de responsabilidades entre classes e objetos. São eles: Strategy Observer, Template Method, Visitor, Chain of Responsibility, Command, Interpreter, Iterator, Mediator, Memento e State.
